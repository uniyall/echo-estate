Per-clip request lifecycle

① Agent uploads clip
Node.jsReceives multipart form (label + MP4/MOV). Streams to R2 via AWS SDK. Creates scene_clip record via Go API (status: queued).
② RunPod dispatch
Node.jsCalls RunPod API to launch RTX 4090 Pod. Passes R2_CLIP_URL + CLIP_ID + STEPS as env vars. Updates clip status to processing.
③ GPU pipeline
RunPodffmpeg → quality gate → COLMAP SfM → quality gate → OpenSplat (10k–30k steps) → upload .ply to R2 → Pod terminates.
④ Polling loop
Node.jsEvery 30s, checks RunPod job status. On completion, retrieves .ply R2 URL. Calls PATCH /clips/:id on Go to mark status ready + store ply_url.
⑤ View Map update
GoClip now has status: ready. Frontend polls GET /clips/:id/status. Agent sees new bubble appear on the View Map canvas, positions it, saves layout.
⑥ Buyer loads splat
GoGET /properties/:id/viewmap returns layout JSON. Buyer clicks hotspot → frontend fetches .ply from R2 signed URL → Svelte + Three.js renders inline.
